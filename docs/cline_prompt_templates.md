# Cline Prompt Templates

以下模板是為這個專案整理的「新對話起手式」。

---

## 1. 通用新任務起手模板

```md
請先閱讀以下文件後再開始：
- `.clinerules`
- `ARCH.md`
- `docs/current_status.md`

先不要改程式。

請先用繁體中文完成以下事情：
1. 總結你對目前專案狀態的理解
2. 指出這次任務最可能涉及的檔案
3. 說明影響範圍與可能風險
4. 提供一個 5~10 點的最小 implementation plan / pseudocode

限制：
- 一次只做這一個原子任務
- 不要順手重構無關程式碼
- 不要改 DSL / execution semantics，除非我明確要求
- `node.data.params` 只放業務資料
- `node.data` 只放 UI state
- 不要使用 `as any`

等我確認方案後，再開始改碼。
```

---

## 2. 方案確認後的實作模板

```md
方案可以，請開始實作。

要求：
- 只改必要檔案
- 維持最小變更
- 若改動會影響其他檔案，請在修改前先說明
- 完成後更新 `docs/current_status.md`
- 若有程式碼變更，請執行：
  - `npm run typecheck`
- 若這次變更可能影響 build 或 editor 整體流程，且成本低，再加跑：
  - `npm run build`

最後請用以下格式回報：
1. 修改了哪些檔案
2. 每個檔案改了什麼
3. 為什麼這樣改
4. 驗證結果
5. 風險 / 後續建議
```

---

## 3. 連續修不好時的中止模板

```md
先停，不要繼續改程式。

請只做以下事情：
1. 總結前面修復失敗的根因
2. 說明目前還不確定的架構點
3. 列出如果繼續修改，最可能踩到的風險
4. 給我一個更小的切分方案

先不要再編輯檔案，等我確認。
```

---

## 4. 適合這個專案的簡短起手式

如果你不想每次打很長，可以直接用這版：

```md
先讀 `.clinerules`、`ARCH.md`、`docs/current_status.md`。
先不要改程式，先用繁體中文總結現況、列出受影響檔案、給我最小 plan。
維持原子任務，不要順手重構；`node.data.params` 是業務資料，`node.data` 是 UI state。
我確認後你再開始改。
```
